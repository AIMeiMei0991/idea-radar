# 创意雷达 UX 改版设计文档

**日期**：2026-04-06  
**范围**：单 HTML 升级（方案 A）— 前端 UX 重构 + 数据质量优化  
**目标**：将"信息墙"体验升级为"创业灵感引擎"，让一人创业者进来就知道该怎么行动

---

## 一、背景与核心问题

**产品定位**：创意雷达 = 传统痛点发现 × 创新解法设计 × 一人公司可行性评估  
**核心用户焦虑**：这个需求我一个人能做吗？

**当前 UX 问题**：
1. 顶栏拥挤：logo + 副标题 + 3 个按钮挤在两行，移动端布局错位
2. 筛选器过载：Tab 栏 + filter chips 两层共 20+ 选项，认知负担极重
3. 卡片信息密度高：每张卡片 8-10 个信息点，扫视疲劳
4. 无二级详情页：点击只是内联展开，无法深度阅读 AI 分析
5. 数据质量差：有 60 条 score≤2 的垃圾数据；98 条评分超出 1-5 范围（系统 bug）

---

## 二、架构决策

**保持单 `public/index.html` 架构**，理由：
- 零迁移风险，GitHub Pages 无需改部署
- 详情页用全屏 overlay + hash URL 实现，体验等同真实路由
- 核心 UX 问题（详情页/筛选器/顶栏）可完全在单文件内解决

---

## 三、前端设计

### 3.1 顶栏重组

**目标**：从 ~72px 压缩到 52px，释放内容空间

```
行1：[🎯 创意雷达]              [日期 ▼]  [⋯]
行2：最后更新：4月6日 16:54（小字灰色）
```

- "清空已读" / "隐藏已读" 合并进右上角 `[⋯]` 下拉菜单
- 副标题"传统需求 × 创新解法 = 一人公司机会"移除（核心用户已知，对新用户无引导作用）
- 日期选择器保留，样式精简为文字按钮

### 3.2 筛选器重构

**Tab 栏**（4 个，删掉来源维度的 Tab）：
```
精选 | 全部 | 验证收入 | 收藏
```
删除：PH新品 / Reddit / 创客 / HN / 中文（来源平台不是用户决策维度，由 chip 过滤覆盖）

**快速筛选 chip**（默认展示 4 个核心，其余折叠）：
```
[ 🇨🇳 中国可做 ]  [ 👤 一人可行 ]  [ ⭐ 高价值 ]  [ ⏱ 快速开发 ]   [更多 ▾]
```
- 激活态保留现有语义色系（绿/紫/橙/蓝）
- "更多"展开后显示：开发周期细分 / 痛点类型 / 需团队

### 3.3 列表卡片精简

每张卡片只展示"决策所需最少信息"：

```
┌────────────────────────────────────────┐
│ Reddit · 效率工具          ★★★★★      │
│                                        │
│ 游戏化任务App：专治拖延上瘾            │
│                                        │
│ 用户打开待办清单后3秒内切出App刷短视频 │  (最多2行，灰色)
│                                        │
│ [👤 一人可行] [🇨🇳 中国可做] [⏱ 1-3月]│
└────────────────────────────────────────┘
```

**删除**：
- 卡片内三区色块展开（痛点/解法/评估）→ 全部移至详情页
- 底部"收藏/分享/原文"操作栏 → 移至详情页
- expand hint 箭头 → 整张卡点击进详情页

**保留**：今日精选横滑 banner（featured card 也精简为标题 + 3 个 pill，点击同样进详情页）

### 3.4 详情分析页（核心新功能）

**触发**：点击任意卡片（列表卡片 + 精选 banner 卡片）  
**实现**：`position:fixed; top:0; left:0; width:100%; height:100%; z-index:500` 的全屏 div，`transform:translateY(100%)` 初始态，点击后 `translateY(0)` 滑入，动画 `0.3s ease-out`  
**URL**：更新为 `#idea-{id}`，刷新后页面检测 hash 自动打开对应详情；分享链接可直接定位  
**交互**：全页滚动（无折叠区块），顶部 sticky 导航栏（高 44px），左划 或 点返回按钮 关闭（`translateY(100%)` 退场）

## 四、数据质量优化

### 4.1 评分 bug 修复

**问题**：分析脚本有时输出 1-10 分制，有 98 条 score > 5  
**修复**：在 prompt 中明确约束 `score 必须是 1-5 的整数`，并在写入前做 clamp：
```js
item.score = Math.min(5, Math.max(1, Math.round(item.score)));
```

### 4.2 质量门槛

**规则**：score < 3 的条目不写入 `ideas.json`（当前有 60 条，约 14%）  
**额外校验**：
- `title` 长度 < 5 或 > 100 → 丢弃
- `problem` 为空 → 丢弃
- `isPainPoint === false` → 丢弃（非痛点内容不符合产品定位）

### 4.3 新增 3 个 AI 分析字段

在 `scrape_and_analyze.js` 的 prompt 中新增要求：

```js
// 新增字段（加入 prompt 要求 AI 输出）
risks: string,       // 2-3 条主要风险，每条一句话
monetization: string, // 变现路径：免费功能/付费分层/预估MRR
marketSize: string   // 受众规模估算，带数字
```

**示例输出**：
```json
{
  "risks": "安卓锁屏权限各品牌差异大需专项适配；国内应用商店上架周期2-4周；已有同类App Forest占据品牌心智",
  "monetization": "免费版基础任务锁屏；Pro ¥18/月含高级统计+无限任务；预估12个月可达¥5000/月MRR（参考Forest中国区定价）",
  "marketSize": "中国职场人+学生约3.2亿，自律工具细分市场约2000万活跃用户"
}
```

## 五、不做的事

- 不迁移到 Next.js / hash router 重构（方案 A 已能覆盖所有 UX 需求）
- 不加社区功能（评论/提交）
- 不做 SEO 优化
- 不改数据抓取来源（另立项）

---

## 六、验收标准

1. 顶栏在 iPhone 14（390px）下单行显示不错位
2. 筛选器默认只展示 4 个 chip，点"更多"才展开
3. 点击任意卡片 → 详情页从底部滑出，URL 变为 `#idea-{id}`
4. 详情页包含：痛点/解法/一人可行性（含 MVP+技术栈+变现+风险）/中国市场 四个区块
5. `ideas.json` 无 score > 5 的数据，无 score < 3 的数据
6. 所有条目有 `risks`、`monetization`、`marketSize` 字段
